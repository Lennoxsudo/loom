多条 tool call 连续出现时：折叠态几乎像消息流里的“行”，不会把对话顶出视口。

改什么

  • 默认折叠：成功后只保留一行命令 + 状态，输出按需展开（现状常占大块高度）。
  • 元信息内联：exit / duration / lines 放进 header，去掉独立 footer 那一层垂直占用。
  • 更轻的表面：折叠时几乎无边框；悬停/展开才出现表面，减少“卡片堆叠”感。
  • 命令展示：折叠单行 ellipsis；展开最多两行，避免 PowerShell 长命令把卡片撑爆。
  • 输出排版：11px mono、line-height 1.45、左缩进对齐 $，默认约 8–10 行后 fade，再 “Show all”。
  • 状态：小圆点 + 数字 exit code，不用粗 pill；running 用点脉冲。
~28px 折叠态行高（目标）
−45–60% 常见成功结果垂直占用
1 行 扫读所需最少信息
0 重写 可落到现有 ExecCommandCard

落地建议（真实组件）

  • ExecCommandCard.tsx：默认 outputExpanded = false（running 时仍自动展开）。
  • ExecCommandCard.module.css：按本 demo 的 .nx* 密度改 padding / type / meta。
  • 折叠态把 exit + duration 移入 header；展开再显示 rail（可选 copy output）。
  • 长命令：折叠 ellipsis，展开 clamp 2 行；完整命令靠 copy。
  • 保持现有 approval footer / pending border 语义，仅收紧视觉。

交互

点击行展开/折叠；悬停显示 copy；展开后可 “Show all” 取消输出截断。本页为纯 HTML 演示，确认后再改仓库组件。
