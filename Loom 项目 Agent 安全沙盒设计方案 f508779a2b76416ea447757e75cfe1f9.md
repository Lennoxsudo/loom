# Loom 项目 Agent 安全沙盒设计方案

<aside>
🧰

一套专为 Loom **项目 Agent（AI 代理）** 设计的纵深防御安全沙盒。目标：把 agent 的一切文件/命令/网络行为都关进「默认拒绝、越界即拦」的边界里，同时不误伤用户自己的手动操作。本方案已对齐 `sandbox.rs / file_ops.rs / terminal.rs / tool_executor.rs` 的真实代码，并参考了 Codex、Claude Code、NVIDIA 的公开沙盒实践。

</aside>

## 1. 目标与威胁模型

**要防的不是「善意 agent 会不会犯错」，而是「当 agent 被劫持或跑飞时，爆炸半径有多大」。**

| 威胁 | 场景 | 现状风险 |
| --- | --- | --- |
| 提示注入（Prompt Injection） | 仓库文件、网页抓取、MCP/工具返回内容里藏指令，劫持 agent | 高：`fetch_web_content` / 读文件内容直接进上下文 |
| 越权读取 / 数据外泄 | agent 读取工作区外的 `~/.ssh`、`.env`、凭据后经网络外传 | 高：**所有读操作零管控**，无 `validate_read` |
| 越权写入 / 破坏 | 写/删/移动到工作区外，或覆盖 `.git` | 中：写有 `validate_write`，但可被符号链接绕过 |
| 任意命令执行 | `execute_command` 跑 `rm -rf`、下载执行 | 中高：命令档位有校验，但 **PTY 终端完全不拦** |
| 沙盒绕过 | 符号链接穿越、走未受管的通道 | 高：路径仅做词法归一化，不解析 symlink |
| 密钥泄露 | agent 进程继承了含密钥的环境变量 | 高：无环境净化 |

> 核心结论：光靠应用层 `if` 校验不够——**必须叠加操作系统内核级强制隔离**，否则任何一处漏挂载点都是逃逸口。这也是 Codex（Seatbelt/Landlock/受限令牌）与 Claude Code（目录+网络边界）的共同做法。
> 

## 2. 设计原则

1. **默认拒绝（default-deny）**：未显式允许的路径/主机/命令一律拒绝，而非黑名单。
2. **纵深防御（defense-in-depth）**：应用层校验 + OS 内核隔离 + 网络代理，任一层失效仍有兜底。
3. **OS 级强制 > 应用层自觉**：真正的边界由内核执行，agent 进程无法自我解除。
4. **最小权限**：只挂载「项目目录」，绝不挂载 home 目录；读写分离授权。
5. **越界即审批**：跨边界动作（网络连接、工作区外写、危险命令）逐次弹审批，禁止「允许一次=永久放行」。
6. **按来源区分**：agent 工具调用 vs 用户手动操作必须可区分，沙盒只约束 agent。
7. **可审计**：每一次拦截/放行/审批都留结构化日志。

## 3. 访问档位模型（Access Tiers）

把当前简单的 `read_only / auto` 扩展为四档，对齐 Codex/Claude 的「模式」理念：

| 档位 | 读 | 写 | 命令 | 网络 | 适用 |
| --- | --- | --- | --- | --- | --- |
| `read_only` | 仅工作区内 | ❌ | ❌ | ❌ | 代码审查、问答 |
| `workspace_write`（默认） | 仅工作区内 | 仅可写根内 | 工作区内、非危险 | ❌（需审批） | 常规编码 |
| `workspace_write_net` | 仅工作区内 | 仅可写根内 | 工作区内 | 允许列表内主机 | 需装依赖/联网构建 |
| `trusted` | 全盘 | 全盘 | 任意 | 任意 | 显式信任，逐次审批高危项 |
- 「危险命令」（`rm -rf /`、指向 home 或系统盘、`git push`）即使在 `workspace_write` 也强制弹审批（Claude Code 同款规则）。
- 网络默认关，开启后走**出口允许列表**，而非全开。

## 4. 分层架构（纵深防御）

### L1 — 应用层策略校验（修补现有 SandboxContext）

补齐当前 `SandboxContext` 的三大空洞：

- **加入 `validate_read`**：读操作也要落在允许读的根内（防 `~/.ssh`、`.env` 外泄）。当前 `read_file_content*`、`open_folder`、`search_in_folder`、`glob_search_files`、`get_file_tree`、`get_file_info` 以及 `tool_executor.rs` 全部读工具都要接入。
- **路径做真实规范化 + symlink 解析**：把 `normalize_path`（仅词法）换成 `canonicalize()` 后再做 `starts_with` 校验，堵住符号链接穿越（这正是 DesktopCommander CVE-2024 的漏洞类型）。
- **落实 `validate_network`**：从占位改为真实出口治理（见 L3）。
- **覆盖 PTY 终端**：`write_to_terminal` 带 `source:"ai"` 时必须先过命令校验，否则 agent 可绕过 `execute_command` 的所有限制直接从 PTY 逃逸。
- **按来源标记**：`SandboxContext` 增加调用来源维度，用户 UI 操作不受档位约束，只有 `source:"ai"` 受约束。

### L2 — 操作系统内核级强制隔离（新增，最关键）

应用层校验只要漏一个挂载点就失守，必须在**子进程 spawn 时**套上内核沙盒。按平台：

| 平台 | 机制 | Rust 落地 |
| --- | --- | --- |
| Windows（Loom 主平台） | 受限令牌（write-restricted token）+ Job Object + 合成 SID `sandbox-write`  • ACL 保护 `.git` | `windows` crate / 参考 Codex Windows 沙盒 |
| Linux | Landlock（文件）+ seccomp（syscall）+ 可选 bubblewrap | `landlock` crate v0.4 |
| macOS | Seatbelt（`sandbox-exec` 配置 profile，`(deny default)`） | 生成 `.sb` profile 后包裹子进程 |
- 关键：沙盒在 `execute_command` / `execute_command_bg` **spawn 子进程前**套用，并**继承到所有孙进程**，agent 无法从内部解除。
- Loom 是 Tauri（Windows 上默认 PowerShell、`C:\` 路径），**优先做 Windows 受限令牌**，再补 Linux/macOS。

### L3 — 网络出口治理（Egress Control）

- 默认**拒绝所有出站**；开启网络档位时走**本地出口代理**，仅放行允许列表内主机（如 [registry.npmjs.org](http://registry.npmjs.org)、[pypi.org](http://pypi.org)、[github.com](http://github.com)）。
- `fetch_web_content` 与命令内发起的连接都要经代理，禁止 agent 直连任意 IP。
- 参考 macOS `agent-seatbelt-sandbox`：一切出站过本地可控代理，内核兜底。

### L4 — 审批门与审计

- **审批门**：跨边界动作（工作区外写、危险命令、每个新网络主机）逐次弹窗；「run-once」不等于「run-many」。
- **审计日志**：结构化记录 `{时间, 来源, 动作, 路径/命令/主机, 判定, 是否审批}`，可复用现有 `OP_HISTORY` 扩展。

### L5 — 密钥隔离与环境净化

- spawn agent 子进程时**清空并重建环境变量**，只注入白名单变量（`terminal.rs` 目前直接继承整个 `PATH`/env）。
- 密钥用「运行时注入」而非环境变量长期暴露；`.env`、凭据目录纳入读拒绝名单。

## 6. 关键代码骨架（Rust 示例）

**6.1 规范化 + symlink 安全的路径校验（替换 `path_within_roots`）**

```rust
fn resolve_safe(path: &Path) -> Result<PathBuf, String> {
    // 对不存在的目标，规范化其最近的已存在父目录，再拼回文件名
    let mut cur = path;
    let mut tail = Vec::new();
    let base = loop {
        match cur.canonicalize() {
            Ok(real) => break real,
            Err(_) => {
                if let Some(name) = cur.file_name() { tail.push(name.to_owned()); }
                match cur.parent() { Some(p) => cur = p, None => return Err("无法解析路径".into()) }
            }
        }
    };
    let mut real = base;
    for name in tail.into_iter().rev() { real.push(name); }
    Ok(real)
}

fn path_within_roots(path: &Path, roots: &[String]) -> bool {
    if roots.is_empty() { return false; } // 默认拒绝，而非放行
    let Ok(real) = resolve_safe(path) else { return false; };
    roots.iter().any(|r| {
        std::path::Path::new(r).canonicalize()
            .map(|root| real.starts_with(root)).unwrap_or(false)
    })
}
```

**6.2 读校验 + 来源区分**

```rust
pub enum CallSource { User, Ai }

impl SandboxContext {
    pub fn validate_read(&self, path: &Path, src: CallSource) -> Result<(), String> {
        if matches!(src, CallSource::User) { return Ok(()); } // 用户手动操作不受限
        if !path_within_roots(path, &self.readable_roots) {
            return Err(format!("读取路径超出允许范围: {}", path.display()));
        }
        Ok(())
    }
}
```

**6.3 Linux Landlock 子进程封装（spawn 前自我限制）**

```rust
use landlock::{Access, AccessFs, PathBeneath, PathFd, Ruleset, RulesetAttr, RulesetCreatedAttr, ABI};

fn lock_fs(read_roots: &[String], write_roots: &[String]) -> Result<(), String> {
    let abi = ABI::V2;
    let mut rs = Ruleset::default().handle_access(AccessFs::from_all(abi))
        .map_err(|e| e.to_string())?.create().map_err(|e| e.to_string())?;
    for r in read_roots {
        rs = rs.add_rule(PathBeneath::new(PathFd::new(r).map_err(|e| e.to_string())?,
            AccessFs::from_read(abi))).map_err(|e| e.to_string())?;
    }
    for w in write_roots {
        rs = rs.add_rule(PathBeneath::new(PathFd::new(w).map_err(|e| e.to_string())?,
            AccessFs::from_all(abi))).map_err(|e| e.to_string())?;
    }
    rs.restrict_self().map_err(|e| e.to_string())?; // 内核强制，进程无法解除
    Ok(())
}
```

**6.4 macOS Seatbelt profile（默认拒绝）**
